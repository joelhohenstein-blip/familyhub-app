## Compression Engine

Module: `app/utils/compressionEngine.ts`

### Functions

#### `initializeCompressionEngine(config?)`

Initialize the compression engine with custom configuration.

**Parameters:**
- `config?: Partial<CompressionEngineConfig>` - Custom configuration options

**Returns:** `void`

**Example:**
```typescript
import { initializeCompressionEngine } from '~/utils/compressionEngine';

initializeCompressionEngine({
  adaptiveCompression: true,
  maxQuality: 95,
  minQuality: 30,
  tolerance: 5,
  cacheResults: true,
  cacheTTL: 24 * 60 * 60 * 1000,
});
```

---

#### `getCompressionEngineConfig()`

Get the current compression engine configuration.

**Returns:** `CompressionEngineConfig`

**Example:**
```typescript
const config = getCompressionEngineConfig();
console.log(config.maxQuality); // 95
```

---

#### `getCompressionEngineMetrics()`

Get compression engine metrics including cache hit rate and average compression time.

**Returns:** 
```typescript
{
  totalCompressions: number;
  totalTime: number;
  cacheHits: number;
  cacheMisses: number;
  averageCompressionTime: number;

}
```

**Example:**
```typescript
const metrics = getCompressionEngineMetrics();
console.log(`Cache hit rate: ${(metrics.cacheHitRate * 100).toFixed(1)}%`);
console.log(`Average time: ${metrics.averageCompressionTime.toFixed(0)}ms`);
```

---

#### `clearCompressionCache()`

Clear all cached compression results.

**Returns:** `void`

**Example:**
```typescript
clearCompressionCache();
```

#### `getCachedCompressionResult(cacheKey)`

Get a cached compression result.

**Example:**

}
```

cacheCompressionResult(cacheKey, result);
```

#### `generateCompressionCacheKey(input, profile, quality?, format?)`

Generate a cache key for compression.

**Parameters:**
- `input: string` - Input file path
- `profile: CompressionProfileName` - Compression profile
- `quality?: number` - Quality level (optional)
- `format?: ImageFormat` - Output format (optional)

**Returns:** `string`

**Example:**
```typescript
const key = generateCompressionCacheKey('photo.jpg', 'mobile', 75, 'webp');
// Returns: "photo.jpg:mobile:75:webp"
```

## Srcset Generator

Module: `app/utils/srcsetGenerator.ts`

### Functions

#### `generateSrcset(options)`

Generate responsive image srcset.

**Parameters:**
- `options: SrcsetGenerationOptions`
  - `basePath: string` - Base image path
  - `widths?: number[]` - Width variants (default: all breakpoints)
  - `dprs?: number[]` - Device pixel ratios (default: [1, 2])
  - `formats?: ImageFormat[]` - Formats (default: ['webp', 'jpeg'])
  - `sizes?: string` - Sizes declaration
  - `useDensity?: boolean` - Use density descriptors
  - `includeSize?: boolean` - Include file size info
  - `namingPattern?: string` - Naming pattern

**Returns:** `SrcsetResult`

**Example:**
```typescript
import { generateSrcset } from '~/utils/srcsetGenerator';

const srcset = generateSrcset({
  basePath: '/images/hero',
  widths: [320, 640, 960, 1280],
  formats: ['webp', 'jpeg'],
  sizes: '(max-width: 640px) 100vw, (max-width: 1024px) 80vw, 60vw',
});

console.log(srcset.srcset);
// "/images/hero-320w.webp 320w, /images/hero-640w.webp 640w, ..."
```

---

## Batch Processor

Module: `app/utils/batchProcessor.ts`

### Classes

#### `BatchProcessor<T, R>`

Generic batch processor for parallel processing.

**Constructor:**
```typescript
new BatchProcessor(
  worker: (item: T) => Promise<R>,
  config?: Partial<BatchProcessorConfig>
)
```

**Methods:**

##### `process(items, onProgress?)`

Process batch of items.

**Parameters:**
- `items: T[]` - Items to process
- `onProgress?: (progress: BatchProgress) => void` - Progress callback

**Returns:** `Promise<{ results: R[]; errors: Array<{ item: T; error: Error }> }>`

**Example:**
```typescript
const processor = new BatchProcessor(
  async (item) => {
    // Process item
    return result;
  },
  { workers: 4 }
);

const { results, errors } = await processor.process(items, (progress) => {
  console.log(`${progress.percentage.toFixed(1)}% complete`);
});
```

---

##### `getMetrics()`

Get batch processing metrics.

**Returns:** `BatchMetrics`

**Example:**
```typescript
const metrics = processor.getMetrics();
console.log(`Success rate: ${metrics.successRate.toFixed(1)}%`);
console.log(`Processing rate: ${metrics.processingRate.toFixed(2)} items/s`);
```

---

##### `getProgress()`

Get current progress.

**Returns:** `BatchProgress`

**Example:**
```typescript
const progress = processor.getProgress();
console.log(`${progress.processed}/${progress.total} items processed`);
```

---

##### `cancel()`

Cancel processing.

**Returns:** `void`

**Example:**
```typescript
processor.cancel();
```

---

#### `CompressionBatchProcessor`

Specialized batch processor for image compression.

**Constructor:**
```typescript
new CompressionBatchProcessor(
  compressionFn: (input: string, output: string, profile: CompressionProfileName) => Promise<CompressionResult>,
  config?: Partial<BatchProcessorConfig>
)
```

**Methods:**

##### `processBatch(items, onProgress?)`

Process batch of images.

**Parameters:**
- `items: Array<{ input: string; output: string; profile: CompressionProfileName }>`
- `onProgress?: (progress: BatchProgress) => void`

**Returns:** `Promise<BatchCompressionReport>`

**Example:**
```typescript
const processor = new CompressionBatchProcessor(
  async (input, output, profile) => {
    // Compress image
    return result;
  }
);

const report = await processor.processBatch(
  [
    { input: 'photo1.jpg', output: 'compressed/photo1.webp', profile: 'mobile' },
    { input: 'photo2.jpg', output: 'compressed/photo2.webp', profile: 'mobile' },
  ],
  (progress) => {
    console.log(createProgressBar(progress));
  }
);
```
