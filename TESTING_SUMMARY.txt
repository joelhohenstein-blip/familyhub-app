================================================================================
                           HOW IT WORKS: TECHNICAL
================================================================================

DATABASE ISOLATION:
  Every table has a "familyId" column
  ├─ posts.familyId
  ├─ media.familyId
  ├─ calendar_events.familyId
  ├─ family_members.familyId
  └─ [ALL OTHER TABLES]

ACCESS CONTROL:
  Every tRPC query verifies TWO things:
  1. Is user a member of the requested family?
  2. Filter results by familyId

FRONTEND:
  familyContext tracks currentFamily
  └─ User can switch families via FamilySwitcher dropdown

RESULT:
  ✓ Smith family sees ONLY Smith data
  ✓ Anderson family sees ONLY Anderson data
  ✓ Johnson family sees ONLY Johnson data
  ✓ Zero cross-family data leakage
  ✓ All 10 families on 1 server, 1 database

For 10 different families wanting their own FamilyHub site:
  
  ❌ DON'"'"'T: Create 10 separate servers and databases
  ✅ DO:    Run them all on 1 deployment with familyId isolation

Benefits:
  ✓ Cost: 1 server instead of 10
  ✓ Maintenance: 1 database instead of 10
  ✓ Scalability: Adding family #11 is one INSERT statement
  ✓ Security: Row-level isolation prevents data leakage
  ✓ User Experience: Family switching via sidebar dropdown

Architecture Pattern: Multi-Tenant SaaS
Isolation Method: Row-Level Security (familyId)
Implementation: React Router + tRPC + Drizzle + PostgreSQL
